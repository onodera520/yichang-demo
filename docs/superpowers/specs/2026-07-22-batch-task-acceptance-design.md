# 任务批量验收设计

## 目标

提高主管处理待验收任务的效率，同时保留系统校验、验收记录和来源异常状态同步。修改范围仅限任务协同页及其验收状态逻辑。

## 单条验收

- 保留员工提交结果、系统验收检查和选填备注。
- 删除“已核对员工处理结果和凭证”复选框。
- 系统检查全部通过时，“验收通过”按钮直接可用；存在未通过项时按钮禁用。
- 点击按钮即代表主管确认验收，仍向状态层传递明确的确认标记。

## 批量验收

- “批量验收通过”按钮仅在“待验收”标签显示。
- 只读取当前筛选结果中被主管勾选的待验收任务，忽略其他标签或不可见结果中的残留选择。
- 点击按钮后展示一次汇总确认，包含已选择、可验收、将跳过的数量，并列出跳过任务及原因。
- 确认时逐条复用单任务验收规则。合格项完成验收，不合格项跳过；单条失败不阻断其他任务。
- 没有合格任务时禁用确认按钮。

## 操作结果

- 每条成功任务独立记录验收人、验收时间、系统检查结果和处理日志，并同步完成来源异常。
- 成功任务从批量选择中移除，跳过任务保持勾选，方便主管继续检查。
- Toast 汇总展示通过和跳过数量。
- 批量验收不触发单任务连续切换意图；列表更新后定位到第一条剩余待验收任务，没有剩余任务时显示队列完成空态。

## 测试范围

- 单条验收不再要求额外勾选，系统检查仍是硬门禁。
- 批量验收只处理当前可见选择。
- 混合选择时合格项通过、不合格项跳过，并返回准确原因。
- 成功项和跳过项的勾选状态按设计更新。
- 验收日志、来源订单或库存状态保持一致。
- 任务协同现有连续处理、筛选、分页和其他批量操作不回归。
