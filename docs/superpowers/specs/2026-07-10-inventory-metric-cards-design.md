# 库存决策指标卡样式同步设计

## 范围

仅修改 `src/pages/Inventory.jsx` 中的 `InventoryMetricCard` 和该页面本地 `Sparkline`。保留库存决策页四列布局、指标数据、筛选、表格、抽屉、路由和“查看详情”交互。

## 视觉规则

- 卡片采用异常工作台指标卡的 `160px` 高度、14px 圆角、柔和边框与阴影。
- 卡片上下各保留 16px 留白，图标为 40px，标题为 17px 中等字重。
- 主数字独立占一行并从卡片左侧开始，使用 30px 字号。
- “较昨日/变化值”和“查看详情”占据完整宽度并两端对齐。
- 折线固定在距卡片底部 16px 的区域，宽度填满内容区，与信息行不重叠。
- 保留红色上涨、绿色下降的现有业务含义，并增加与折线色一致的末端圆点。

## 实现边界

- 不抽取或修改公共 `MetricCard`。
- 不修改异常工作台及其他页面。
- 不改变 `metricCards` 数据结构或点击处理逻辑。
- 不引入新依赖。

## 验证

- 增加库存指标卡布局回归测试，锁定高度、留白、全宽信息行、折线位置和末端圆点。
- 运行现有逻辑测试、`npm run build` 和 `git diff --check`。
- 构建通过后覆盖部署到当前 Vercel production，并确认状态为 `Ready`。
