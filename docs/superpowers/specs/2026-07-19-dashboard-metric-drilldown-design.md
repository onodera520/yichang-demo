# 异常工作台指标卡真实数据与下钻设计

## 目标

将异常工作台五张指标卡从固定展示数字和代表性单条详情，改为可复算的实时聚合指标。卡片当前值必须来自 `DemoStateContext` 中的订单与库存数据，并与点击后的落地列表口径一致。

## 统计口径

“未解决订单”指状态不为 `已完成`、`已驳回` 的订单；“未解决库存风险”指状态不为 `已完成` 的 SKU。

| 指标 | 当前值 | 落地集合与排序 |
| --- | --- | --- |
| 高风险订单 | 未解决且风险等级为 `高` 的订单数 | `/orders`，高风险 + 未解决，剩余 SLA 升序 |
| 即将缺货SKU | 未解决且可售天数不超过 7 天的 SKU 数 | `/inventory`，7 天内缺货 + 未解决，可售天数升序 |
| 物流延误 | 未解决且异常类型为 `物流延误` 的订单数 | `/orders`，物流延误 + 未解决，剩余 SLA 升序 |
| 售后高发 | 未解决且异常类型为 `退款` 的订单数 | `/orders`，退款 + 未解决，剩余 SLA 升序 |
| 潜在亏损 | 未解决高风险订单的 `amount` 合计 | `/orders`，高风险 + 未解决，影响金额降序 |

平台和店铺全局筛选继续参与订单指标统计；库存指标参与平台筛选。搜索关键字不改变指标卡，避免输入搜索时整组卡片频繁跳变。

## 数据与交互

- 新增无副作用的指标聚合函数，统一输出 `value`、格式化类型、昨日差值和趋势数据；工作台只消费聚合结果，不再引用 `dashboardStats` 的固定当前值。
- 历史趋势存放为同口径的每日聚合快照；折线使用历史快照加实时当前值，昨日变化由实时当前值减去最近一日快照计算。历史仍属于 Mock 数据，但每个显示值均有明确来源，不再维护孤立数字。
- 五张卡片整卡可点击，并支持 Enter/Space。原“查看详情”保持视觉不变，但不再是唯一热区。
- 路由状态使用明确的 `dashboardPreset`，订单与库存页解析预设后设置筛选、排序和“仅未解决”条件。落地页显示可移除的工作台预设提示，防止隐藏筛选。
- 从其他入口进入订单或库存页时保持现有默认行为；抽屉打开、跨页面任务和全局顶部筛选逻辑不变。

## 验证

- 纯函数测试覆盖五项统计、货币合计、未解决状态排除、平台/店铺筛选、输入不可变和历史差值。
- 路由测试验证五张卡片映射到正确预设，订单和库存页能恢复筛选及排序。
- 交互测试验证整卡键盘可用，落地列表数量与卡片一致，清除预设后恢复普通列表。
- 回归任务完成回写：订单完成后相关订单指标和潜在亏损下降；库存任务完成后即将缺货 SKU 数下降。
- 运行全部 `.test.mjs`、`npm run build`、`git diff --check`，并在 1440×900 验证无布局变化。

## 边界

- 不接入真实后端，所谓“真实”指 Mock 数据内部可复算、跨页面一致、随前端状态实时变化。
- 不新增页面、不改变卡片视觉、不重构全局筛选。
- 本次不调整其他页面已有的独立指标卡统计口径。
