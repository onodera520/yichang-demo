# 库存详情抽屉信息压缩设计

## 目标

缩短库存详情抽屉的主要操作路径。用户打开抽屉后，应在首屏或一次很短的滚动内完成 AI 建议采纳、负责人选择和任务创建，同时仍可向下查看完整判断依据、计算口径与销售趋势。

## 信息结构

抽屉内容按以下顺序排列：

1. 数据新鲜度提示，仅在数据异常或连接存在风险时显示。
2. 商品概览，将商品图片、名称、平台、仓库和四项库存指标整合为一个紧凑区块。
3. AI 建议与执行入口，在同一卡片内展示建议方案、建议理由、置信度、处理状态、采纳按钮和负责人选择。
4. 调整方案，默认收起，仅在用户点击底部“调整方案”后展开。补货展示数量、原因和备注；调拨额外展示调出仓与调入仓；滞销不展示补货数量。
5. 建议依据，将 AI 判断依据和库存计算口径整合为一个可展开区块，默认只显示摘要。
6. 销售趋势，移动到建议依据之后并压缩高度，保留完整图表能力。

底部操作栏继续固定，保留“调整方案”和场景对应的创建任务按钮。

## 交互规则

- AI 建议卡中的“采纳”直接复用现有采纳逻辑，不自动创建任务。
- 负责人选择紧邻采纳入口；采纳后可立即分派，不要求先打开调整表单。
- 点击“调整方案”只控制调整表单展开，并保留当前负责人、已填数量、仓库、原因和备注。
- 创建任务继续沿用现有校验：方案必须已保存或采纳、负责人已分派、无未完成重复任务。
- 补货、滞销和调拨三类场景保持各自现有字段、文案和任务类型。
- 建议依据和销售趋势不参与任务创建校验，折叠状态不影响数据。

## 视觉规格

- 商品概览由原来的商品卡加四个独立指标卡，改为单卡布局；指标采用四列或紧凑两行展示。
- AI 建议卡删除置信度进度条，仅保留百分比数值。
- 区块间距维持 12px，内部垂直间距压缩到 8px。
- 销售趋势图高度由 132px 调整到约 88px。
- 可展开区块使用清晰的标题、摘要和展开图标，不增加新的弹窗。

## 状态与可访问性

- 调整方案和建议依据的展开按钮使用原生 `button`，提供 `aria-expanded`。
- 抽屉切换 SKU 时，调整表单恢复该 SKU 的已保存状态并默认收起。
- 已采纳、任务已创建和禁用状态继续使用现有状态源，避免创建另一套页面状态。
- 键盘用户可以按顺序访问采纳、负责人、展开详情和底部操作按钮。

## 验证

- 1440×900 下验证常规补货 SKU 打开后无需长距离滚动即可看到采纳和负责人。
- 验证补货、滞销、调拨三种场景的字段和创建任务逻辑不回归。
- 验证“调整方案”展开后负责人不清空，保存后仍可创建任务。
- 验证建议依据折叠、展开和销售趋势渲染正常。
- 运行库存抽屉相关测试、完整 `node --test` 和 `npm run build`。
